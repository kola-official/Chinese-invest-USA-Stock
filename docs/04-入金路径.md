# 入金路径

入金不是单纯把钱汇出去，而是让资金以“同名、可追溯、可解释”的方式到达券商。第一次应先小额试转。

## 路径总览

| 方案 | 链路 | 成本 | 稳定性 | 适合谁 | 风险点 |
| --- | --- | --- | --- | --- | --- |
| A | 内地银行 -> 香港银行 -> 所选券商 | 中 | 较高 | 已有港卡的新手 | 香港账户属于 CRS 地区，电汇费用可能较高 |
| B | 内地银行 -> 香港银行 -> Wise -> 所选券商 | 低到中 | 中 | 想降低换汇/转账成本的人 | Wise 风控、资料补充、信息交换 |
| C | 内地银行 -> 香港银行 -> 美国银行 -> 所选券商 | 中 | 较高 | 已有美国银行能力的人 | 步骤多，银行问询更多 |
| D | 内地银行 -> 美国银行 -> 所选券商 | 中 | 中 | 有稳定美国银行账户的人 | 境内汇款用途审核 |
| E | 内地银行 -> Wise -> 所选券商 | 中 | 不稳定 | 无港卡无美卡、只想小额验证的人 | 成功率不可复制，不适合大额 |
| F | 加密/金融科技账户 -> 所选平台或券商 | 不确定 | 不稳定 | 已熟悉加密资产的人 | 平台、政策、链上资金来源和 KYC 风险 |

## 港卡直入

流程最直观：

1. 在券商端获取电汇信息，必要时先创建入金通知。
2. 用本人香港银行账户发起汇款。
3. 准确填写收款银行、收款账户、附言、个人券商账号等信息。
4. 到账后保存银行回单和券商入账记录。

这条路适合建立第一条可用通道。缺点是费用和汇率不一定最优，且香港账户的年度信息可能被交换。

## 港卡 + Wise

```mermaid
flowchart LR
  A["内地银行购汇"] --> B["香港银行"]
  B --> C["Wise"]
  C --> D["所选券商"]
```

优势是换汇和本地转账体验较好，部分券商也能通过 Wise 或 ACH 类能力入金。缺点是 Wise 对资金来源、频率、收付款对象非常敏感，不适合把大额资金长期放在里面。

Wise 开户时建议先用身份证完成注册，再开设港元账户。港元账户开通后，通常等待约 30 天左右，再联系 Wise 客服申请把姓名改为英文拼音，并按要求提供护照材料。后续资金在 Wise、银行和券商之间流转时，英文姓名一致会更稳。

## 美国银行中转

```mermaid
flowchart LR
  A["内地或香港银行"] --> B["美国银行"]
  B --> C["所选券商 ACH/电汇"]
```

如果已经有美国银行账户，这通常是更适合长期维护的美元通道。需要提前确认券商是否允许绑定该银行、出金是否必须回到曾经入金的同名账户、银行是否接受券商 ACH 往来。

## 无港卡无美卡时

可以研究 Wise 或其他中转账户的小额路径，但不要预设成功率。不同银行、币种、时间窗口和账户资料都会影响结果。建议把它当作备用验证方案，而不是主要资金通道。

## 加密或特殊平台

只适合已经了解稳定币、链上转账、交易所 KYC 和平台风控的人。要重点检查：

- 平台是否仍接受中国大陆居民。
- 证券业务和加密业务分别由哪个实体运营。
- 是否参与 CRS、CARF 或当地类似信息报告制度。
- 稳定币来源能否解释。
- 出金路径是否清楚，失败时能否申诉。

## 入金前检查表

| 检查项 | 说明 |
| --- | --- |
| 同名 | 银行、Wise、券商姓名拼写尽量一致 |
| 收款信息 | 账号、银行代码、SWIFT/ABA、地址、附言逐项核对 |
| 入金通知 | 有些券商要求先报备金额和来源 |
| 首笔金额 | 先小额，不要第一次就大额 |
| 回单 | 汇出、换汇、到账记录都保存 |
| 退路 | 退汇时资金回到哪里、费用由谁承担 |
